## 概要

以下の環境でIntersection Observerを使って画像を遅延読み込みします。

- Nuxt.js (Composition API)
- TypeScript

Vue.js製の遅延読み込みコンポーネントを提供する [ライブラリ](https://github.com/heavyy/vue-intersect) はすでに存在しますが、このコンポーネントをNuxtプロジェクトで使おうとしたところ以下のエラーが発生したため、Nuxtで実行できる遅延読み込みコンポーネントを作成しました。

> SyntaxError
> 
> Cannot use import statement outside a module

## 挙動の確認

遅延読み込みさせたい要素をObserverコンポーネントのスロットに渡します。サンプルではthresholdに `[0.2, 0.8]` を設定しており、ブラウザのビューポートと画像要素の交差率が0.2または0.8を上回ったり下回ったりするときに `onEnter` や `onLeave` 関数が実行されます。

遅延読み込み前はsrc属性に設定したダミー画像が表示され、`onEnter` が実行されるタイミングでsrcの値をdata-srcの値で書き換えます。

```vue[components/LazyImage.vue]
<template>
  <Observer :threshold="[0.2, 0.8]" @enter="onEnter" @leave="onLeave" @change="onChange">
    <img ref="lazyImageRef" src="/images/dummy.png" :data-src="src" :height="height" :width="width" :alt="alt" :title="title" />
  </Observer>
</template>

<script lang="ts">
import { defineComponent, ref } from '@nuxtjs/composition-api'
// TODO: Polyfill

export default defineComponent({
  props: {
    src: {
      type: String,
      required: true,
    },
    height: {
      type: Number,
      required: false,
    },
    width: {
      type: Number,
      required: false,
    },
    alt: {
      type: String,
      required: false,
    },
    title: {
      type: String,
      required: false,
    },
  },
  setup() {
    const lazyImageRef = ref<HTMLImageElement>()
    return {
      lazyImageRef,
    }
  },
  methods: {
    onEnter() {
      if (!this.lazyImageRef) return
      console.log('onEnter')

      if (this.lazyImageRef.dataset.src) {
        this.lazyImageRef.src = this.lazyImageRef.dataset.src
        delete this.lazyImageRef.dataset.src
      }
    },
    onLeave() {
      console.log('onLeave')
    },
    onChange() {
      console.log('onChange')
    }
  }
})
</script>
```

## パフォーマンス比較

遅延読み込みの有無でLighthouseの各種スコアがどれくらい変化するのか検証しました。スコアの計測条件は以下です。

- 計測対象のページで画像を85枚表示する
- ChromeのDevToolsでLighthouseを実行する
  - Clear Storage, Simulated throttlingにチェックあり
- 対象デバイスはPCSPとする
- 5回計測した平均値を比較する

### 考察

- PCではほとんど差が出ない。原因不明なので要調査。
- SPでは遅延読み込みするとSpeed Indexの値が向上する。それに比例してPerformanceスコアの値も向上する。
  - ただし、Intersection Observerのスクリプトを実行した影響でTotal Blocking Timeの値が増加している。画像数を増やすとTotal Blocking Timeの値も比例して増加する可能性があるので注意する。

## IE対応について

Intersection Obserber APIはIEで対応されていないので利用する際はPolyfillを導入してください。
